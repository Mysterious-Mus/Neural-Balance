# Neural-Balance Experiments

This repository supports folder-based experiment runs for MNIST + FCN.

## Experiment Workflow

Each experiment lives in its own folder, for example:

`experiments/smoke_mnist_fcn/`

Recommended contents:

- `config.yaml`: full run configuration
- `logs/`: outputs generated by the run (`train.csv`, `summary.json`, etc.)

## Run an Experiment

From repo root:

`python -m src.experiments.mnist_fcn.launch_from_config experiments/smoke_mnist_fcn`

Or from inside an experiment folder:

`python -m src.experiments.mnist_fcn.launch_from_config .`

## Config File

Create `config.yaml` in your experiment directory. Minimal example:

```yaml
epochs: 100
lr: 0.001
model: medium_fcn
dataset: mnist
gpu: "0"
batchsize: 256
seed: 0

activation: relu
tanh_on_output: false
lr_scheduler: none

l2_weight: 0.0
order: 2
do_neural_balance: false
neural_balance_epoch: 1  # only used when do_neural_balance is true
full_balance_at_start: false
train_data_frac: 1.0
reverse_balance_layer_order: false

target_tau: 95.0
```

Notes:

- `data_root` is optional; default is `<repo>/data`.
- `metrics_csv` is optional; default is `<experiment_dir>/logs/train.csv`.
- `summary_json` is optional; default is `<experiment_dir>/logs/summary.json`.
- `target_tau` is optional; default is `95.0` (test accuracy \%, used for `epochs_at_tau` in `summary.json`). The legacy key `convergence_tau` is accepted as an alias.
- Boolean flags use `true`/`false` (or `0`/`1`): `do_neural_balance`, `full_balance_at_start`, `reverse_balance_layer_order`, `tanh_on_output`. Legacy keys `neural_balance`, `reversed`, and `neuralFullBalanceAtStart` / `neural_full_balance_at_start` still work.

## Logged Outputs

### `train.csv` (per epoch)

- `epoch`
- `train_accuracy`
- `train_loss`
- `train_eval_loss`
- `test_accuracy`
- `test_loss`

### `summary.json` (per run)

Includes table-aligned aggregate fields:

- `epochs_at_tau`
- `final_train_acc_pct`
- `final_train_loss_eval`
- `final_test_acc_pct`
- `final_test_loss`
- `best_test_acc_pct`
- `generalization_gap_train_minus_test_ppts`
- plus metadata such as `seed` and `target_tau_test_acc_pct`

## Refresh Summary Without Retraining

If you change `target_tau` after a run, you can recompute `epochs_at_tau` from
existing `logs/train.csv`:

`python -m src.experiments.mnist_fcn.refresh_summary_from_logs experiments/your_experiment_dir`

This reads `config.yaml` (including `target_tau`) and rewrites `logs/summary.json`
without rerunning training.
