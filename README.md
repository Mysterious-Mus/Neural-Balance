# Neural-Balance Experiments

This repository supports folder-based experiment runs for MNIST + FCN.

## Experiment Workflow

Each experiment lives in its own folder, for example:

`experiments/smoke_mnist_fcn/`

Recommended contents:

- `config.yaml`: logged snapshot of the configuration that was used
- `logs/`: outputs generated by the run (`train.csv`, `summary.json`, etc.)

## Run an Experiment

From repo root:

`python -m src.experiments.mnist_fcn.launch_from_config experiments/smoke_mnist_fcn --config-path configs/mnist_fcn_regscomp/plain.yaml`

Or from inside an experiment folder:

`python -m src.experiments.mnist_fcn.launch_from_config . --config-path configs/mnist_fcn_regscomp/plain.yaml`

## Config File

Keep authoritative configs under `configs/` and load them with `--config-path`.
The runner copies the loaded config into `<experiment_dir>/config.yaml` for logging/reproducibility.

Minimal base config example (`configs/.../*.yaml`):

```yaml
epochs: 100
lr: 0.001
model: medium_fcn
dataset: mnist
gpu: "0"
batchsize: 256
seed: 0

activation: relu
tanh_on_output: false
lr_scheduler: none

l2_weight: 0.0
order: 2
do_neural_balance: false
neural_balance_epoch: 1  # only used when do_neural_balance is true
full_balance_at_start: false
train_data_frac: 1.0
reverse_balance_layer_order: false

target_tau: 95.0
```

Notes:

- `data_root` is optional; default is `<repo>/data`.
- `metrics_csv` is optional; default is `<experiment_dir>/logs/train.csv`.
- `summary_json` is optional; default is `<experiment_dir>/logs/summary.json`.
- `target_tau` is optional; default is `95.0` (test accuracy \%, used for `epochs_at_tau` in `summary.json`). The legacy key `convergence_tau` is accepted as an alias.
- Boolean flags use `true`/`false` (or `0`/`1`): `do_neural_balance`, `full_balance_at_start`, `reverse_balance_layer_order`, `tanh_on_output`. Legacy keys `neural_balance`, `reversed`, and `neuralFullBalanceAtStart` / `neural_full_balance_at_start` still work.
- `--config-path` accepts repo-relative paths (recommended, e.g. `configs/...`) or absolute paths.

## Logged Outputs

### `train.csv` (per epoch)

- `epoch`
- `train_accuracy`
- `train_loss`
- `train_eval_loss`
- `test_accuracy`
- `test_loss`

### `summary.json` (per run)

Includes table-aligned aggregate fields:

- `epochs_at_tau`
- `final_train_acc_pct`
- `final_train_loss_eval`
- `final_test_acc_pct`
- `final_test_loss`
- `best_test_acc_pct`
- `generalization_gap_train_minus_test_ppts`
- plus metadata such as `seed` and `target_tau_test_acc_pct`

## Refresh Summary Without Retraining

If you change `target_tau` after a run, you can recompute `epochs_at_tau` from
existing `logs/train.csv`:

`python -m src.experiments.mnist_fcn.refresh_summary_from_logs experiments/your_experiment_dir`

This reads `config.yaml` (including `target_tau`) and rewrites `logs/summary.json`
without rerunning training.

## BO Meta Experiments (Regularization Tuning)

You can run a folder-based "meta experiment" for Bayesian optimization (GP + EI)
over regularization hyperparameters while reusing the base training config.

Example authoritative meta config:

`configs/mnist_fcn_regscomp/meta_bo_l2.yaml`

Run from repo root:

`python -m src.experiments.mnist_fcn.run_bo_meta_experiment experiments/mnist_fcn_regscomp_meta_bo_l2 --config-path configs/mnist_fcn_regscomp/meta_bo_l2.yaml`

Dry-run (only logs resolved meta config into experiment dir):

`python -m src.experiments.mnist_fcn.run_bo_meta_experiment experiments/mnist_fcn_regscomp_meta_bo_l2 --config-path configs/mnist_fcn_regscomp/meta_bo_l2.yaml --dry-run`

How it works:

- Loads fixed training settings from `base_config` when present; otherwise from `base_config_path`.
- Creates one concrete experiment folder per trial/seed and launches training through `launch_from_config`.
- Evaluates each BO candidate across a fixed seed list.
- Optimizes `mean_final_val_acc_pct` across seeds with Gaussian Process + Expected Improvement.
- Can train seed runs in parallel using `runtime.parallel_seeds` and assign GPUs with `runtime.gpu_pool`.

When the meta runner logs `<meta_experiment_dir>/config.yaml`, it writes a
self-contained `base_config` block (resolved) to make manual tweaking easier.

Outputs in the meta experiment folder:

- `bo_summary.json` (best trial and full BO settings)
- `bo_trials.csv` (all evaluated candidates + aggregate metrics)
- `trials/trial_*/seed_*/train.csv` and `summary.json` (per-seed run logs)

Note: `configs/mnist_fcn_regscomp/*.yaml` are wrapper profiles that consume
values from `configs/experiment_config.yaml` (`_consume` + `_select`), so the
single source of truth stays in one file.

## Report Sync Workflow (Single Source of Truth)

To reduce manual sync work between experiments and report text, this repo uses
`configs/experiment_config.yaml` as a shared source for key constants.

- Config file: `configs/experiment_config.yaml`
- Macro generator: `scripts/generate_report_macros.py`
- Generated TeX macros: `REPORT/generated/report_values.tex` (auto-generated)
- One-command update script: `scripts/update_report.sh`

### What gets synced

The generator writes LaTeX macros such as:

- `\ValSplitPct`, `\SeedList`, `\NumSeeds`
- `\BOBudgetTotal`, `\BOBudgetInit`, `\BOBudgetGuided`
- `\MaxEpochs`, `\LearningRate`, `\BatchSize`
- `\TargetTauPct`, `\MnistTrainSize`, `\MnistTestSize`

`REPORT/main.tex` includes this generated file via:

`\\input{generated/report_values}`

### Recommended usage

From repo root:

1. Edit `configs/experiment_config.yaml`
2. Run:

`bash scripts/update_report.sh`

This will:

1. regenerate `REPORT/generated/report_values.tex`
2. compile the report (`pdflatex + bibtex + pdflatex + pdflatex`)

If you run the generator directly, use:

`python3 scripts/generate_report_macros.py`

### Notes

- Keep `REPORT/generated/report_values.tex` as generated output; do not edit by hand.
- Tables are intentionally unchanged for now; this workflow currently syncs
  report constants/macros first.
